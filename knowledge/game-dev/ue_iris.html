UE Iris 现代化数据驱动同步系统

Epic 从头重写的现代同步框架。用数据驱动的 FReplicationStateDescriptor 和 ChangeMask 位图取代运行时反射，配合可插拔的 Filter/Prioritizer 和独立 DataStream 管理，实现更快更灵活的同步。

核心概念：数据驱动 vs 对象驱动

Iris 与原生方案的根本差异：原生围绕 UActorChannel（对象）构建，Iris 围绕 ReplicationState（数据描述）构建。这带来了架构层面的灵活性。

1. BeginReplication - 注册入口

游戏侧调 Bridge->BeginReplication 注册要同步的 UObject。返回 FNetRefHandle——Iris 内部的轻量句柄，替代 UActorChannel。

Runtime/Experimental/Iris/Core/Public/Iris/ReplicationSystem/ObjectReplicationBridge.h:111

2. FReplicationStateDescriptor - Iris 的核心创新

类初始化时通过 ReplicationStateDescriptorBuilder 一次性从反射构建：

  • 字段 offset / size
  • 每字段的 FNetSerializer 指针（PackedVector, Quat, Struct...）
  • ChangeMask 位号
  • Push Model 支持标记

后续所有 diff/序列化都是 O(1) 查表，不再运行时反射。

Runtime/Experimental/Iris/Core/Private/Iris/ReplicationState/ReplicationStateDescriptorBuilder.cpp

3. MarkDirty + ChangeMask (Push Model 增量位图)

每个被同步的对象持有一份 ChangeMask（位图）。属性 setter 或 MARK_PROPERTY_DIRTY 宏把对应 bit 置 1。PreSendUpdate 时只需扫 ChangeMask != 0 的对象和位——比原生的"逐字段 diff Shadow State"快得多。

4. PreSendUpdate(DeltaSeconds) - 每帧核心入口

子步骤：

  1. Poll：采样非 Push Model 属性到 State Buffer（ObjectPollFrequencyLimiter）
  2. Filter：通过 NetObjectFilter（空间/兴趣/连接白名单）
  3. Prioritize：通过 NetObjectPrioritizer 打分排序
  4. 生成本帧要发的对象集合
Runtime/Experimental/Iris/Core/Public/Iris/ReplicationSystem/ReplicationSystem.h:101

5. 可插拔过滤器与优先级

NetObjectFilter

类似 RepGraph Node 的现代版。可实现自定义 Filter：空间距离 Filter、队伍/仇恨 Filter、视锥 Filter。返回一个位图指定哪些对象对哪些 Connection 可见。

Runtime/Experimental/Iris/Core/Public/Iris/ReplicationSystem/Filtering/NetObjectFilter.h

NetObjectPrioritizer

每 Connection 每帧对候选对象打分。类似原生 NetPriority，但更灵活：可自定义 SpaceUsage / DistanceSq / RecentActivity 组合。带宽满时截断。

Runtime/Experimental/Iris/Core/Public/Iris/ReplicationSystem/Prioritization/NetObjectPrioritizer.h

6. ReplicationWriter - 脱离 ActorChannel

脱离 UActorChannel/FOutBunch。直接对每个 dirty state 用其 FNetSerializer 序列化到 FNetBitStreamWriter。写入格式包含：Handle 索引 + ChangeMask 位段 + 数据。整体 packet 由 DataStreamManager 组装。

Runtime/Experimental/Iris/Core/Private/Iris/ReplicationSystem/ReplicationWriter.cpp

7. DataStreamManager - 分层数据流

Iris 独立的分层数据流管理。多路复用不同类型流：

  • ReplicationDataStream：属性同步
  • AttachmentReplication：RPC
  • NetTokenDataStream：字符串/引用

每个流独立可靠性/优先级。

Runtime/Experimental/Iris/Core/Private/Iris/DataStream/DataStreamManager.cpp

8. 客户端：ReplicationReader

客户端对称的读取端。用同一份 FReplicationStateDescriptor 反向查表，DequantizeAndApplyHelper 把量化后的字节流解码回 UObject 属性，触发 RepNotify 回调。

9. PostSendUpdate

发完清理：已发出的 dirty bit 移入"pending ACK"记录 -> ACK 到达真正清 mask -> 丢包重新标 dirty。基于 PacketNotification 系统。

Iris 相对原生的优势

  1. 无运行时反射：Descriptor 编译期建好
  2. Push Model：ChangeMask 位图 vs 原生 diff Shadow
  3. 可插拔过滤/优先级：类似 RepGraph 但更细粒度
  4. 脱离 ActorChannel：不再受 128 通道限制
  5. 多线程友好：Descriptor 只读、Poll/Filter 可并行
  6. 更小的 Bit 数：NetSerializer 支持 range/quantization
