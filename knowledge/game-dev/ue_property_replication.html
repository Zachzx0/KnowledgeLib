UE 属性同步 (Property Replication) 完整数据流

从 UPROPERTY(Replicated) 声明到客户端 OnRep_XXX 回调的完整链路。核心机制：FRepLayout 预生成布局 -> DiffProperties 对比 Shadow State -> SendProperties 只写脏字段 -> 客户端反序列化直接写内存。

1. 属性声明

UPROPERTY(Replicated) 或带 RepNotify 的 ReplicatedUsing=OnRep_XXX。还需在 GetLifetimeReplicatedProps 里注册：

DOREPLIFETIME(AMyActor, HP);
DOREPLIFETIME_CONDITION(AMyActor, Gold, COND_OwnerOnly);

2. FRepLayout 初始化（一次性）

类首次被同步时通过 InitFromClass 构建，缓存永久。扫描所有 UPROPERTY(Replicated)：

  • 生成 FRepLayoutCmd（含 offset/size/property/CmdType）
  • 展开数组和嵌套结构
  • 建立 handle 编号

之后每帧 diff 都用这份 Layout 表。

Runtime/Engine/Private/Net/RepLayout.cpp

3. 服务器属性变更

只有服务器有权改——客户端改了会被下一次同步覆盖。Push Model 场景需要额外调 MARK_PROPERTY_DIRTY_FROM_NAME 通知系统。

4. 主循环：ServerReplicateActors

服务器每帧 Tick -> UNetDriver::TickFlush -> ServerReplicateActors。默认约 30Hz（NetServerMaxTickRate）。

BuildConsiderList 筛选 NextUpdateTime 到期的 Actor。Dormant 的 Actor 不进列表（除非 FlushNetDormancy 唤醒）。

5. 每连接 IsRelevant + PrioritizeActors

对每个连接：

  1. 调 Actor->IsNetRelevantFor(ViewerPawn, ViewerLoc) 剔除不相关
  2. 计算优先级 = NetPriority x TimeSinceLast x 距离衰减
  3. 按优先级排序，按带宽预算截取前 K 个

6. ReplicateActor 流程

  1. 首次 -> 序列化 Actor 类 GUID + Location（生成通道）
  2. 收集 Actor 及其所有 ReplicatedSubObjects（Component）
  3. 每个 UObject 一次 FObjectReplicator::ReplicateProperties
  4. 结果拼进一个 Bunch -> SendBunch

7. DiffProperties - 脏字段检测

每个 UObject 的 FObjectReplicator 保存一份 FRepShadowDataBuffer——上次成功发送的所有属性值副本。每帧 diff：逐 FRepLayoutCmd 比对当前值 vs Shadow -> 生成 ChangedPropertyTracker（存 dirty handle 列表）。

这是原生方案最贵的一步——O(属性数)。Iris 的 Push Model 就是为消灭它。

8. SendProperties - 只写脏字段

对每个 dirty handle 写：

  • handle 编号（VarInt 压缩）
  • 值（用属性对应的 NetSerialize）

Vector 走 SerializeCompressedVector（16-bit 量化）；Rotator 压缩到 8-bit。末尾写 Terminator (0)。

9. 发包

所有 dirty 属性 + SubObject 的 Bunch 拼成一个 FOutBunch。大 Bunch 自动切成 PartialBunch（bPartial=1）。可靠 Bunch 存 OutRec 等 ACK，丢包重发。

10. 客户端反序列化 + OnRep 回调

客户端读 handle 循环：

  • 读 handle 编号（0=终止）
  • 用同一份 FRepLayoutCmd 找到属性 offset
  • NetSerialize 反序列化
  • 直接写目标 UObject 内存
  • 触发 OnRep_XXX 回调
即使值和之前一样也会触发（除非用 REPNOTIFY_OnChanged）。OnRep 参数可拿到旧值。

关键优化：Shadow State + Handle 编号

没变的属性完全不写字节；变了也只写 handle 而非属性名。

早期 UE3 用属性名 hash 作为 tag——很浪费带宽。UE4/5 引入 handle 编号（同一 UClass 服务器和客户端 Layout 一致，编号相同）-> 每字段只花 1-2 字节 handle + 值。

Iris 更进一步：连 handle 都不写，直接靠 ChangeMask 位号定位。
