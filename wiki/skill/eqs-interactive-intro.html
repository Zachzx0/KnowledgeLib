1. 一句话定位

`EQS` 是 Unreal AI 系统里“收集环境候选项，并用测试选出最佳项”的查询工具。它常被 `Behavior Tree` 调用，结果通常写入 `Blackboard Key`，再驱动移动、面向目标或后续攻击行为。

?
WHAT
它是什么
一个环境查询资源：生成位置或 Actor 作为 `Item`，再通过测试过滤与计分，返回最适合的问题答案。
AI
WHY
为什么需要
当 AI 不只是“去固定点”，而是要“找最近补给、找掩体、找能看见玩家的攻击点”时，需要把空间选择显式建模。
≠
NOT
它不是什么
它不是 `Behavior Tree` 的替代品，也不是寻路系统本身；它给出候选结果，是否移动、如何移动仍由行为树和导航系统执行。
事实来源：Epic 官方 `Environment Query System`、`Overview` 与 `User Guide` 文档说明 `EQS` 用于收集环境数据，通过 `Generator`、`Test`、`Context` 返回最佳 `Item`，并可从 `Behavior Tree` 调用。

3. 核心词汇：先把节点名字记成动作

Q
Asset
`Environment Query`
查询资源。你在内容浏览器创建它，在 EQS 编辑器中连接 `Root`、`Generator` 和 `Tests`。
G
Generate
`Generator`
产生候选 `Item`。例如 `Points: Grid` 生成一组位置；`Actors of Class` 在范围内找指定类 Actor。
I
Candidate
`Item`
被测试的候选项，可以是位置 `FVector` 或 Actor。调试时常以球体显示。
C
Reference
`Context`
测试和生成器的参考框架。内置有 `Querier`、`Item`，也可用 `EnvQueryContext_BlueprintBase` 或 C++ 自定义。
T
Ask
`Test`
向候选项提问：能否看见玩家？距离多远？路径是否存在？测试可过滤、计分，或二者兼具。
R
Return
`Run Mode`
`Run EQS Query` 可选 `Single Best Item`、前 5% 随机、前 25% 随机或 `All Matching`。
容易混淆：官方文档说明，对同一个 `Generator` 添加多个 `Test` 是常见做法；测试顺序不决定最终语义，过滤测试会先于计分测试执行，以减少后续计分成本。

5. 最小可跑配方：用官方快速入门拆成 6 步

这不是完整项目搭建指南，而是把 `EQS_FindPlayer` 的关键节点压缩成可记忆配方。

启用与资源

  1. 在 `Project Settings > Plugins` 启用 `Environment Query Editor`。
  2. 创建 `Blackboard`、`Behavior Tree`、`Environment Query`、`AIController`。
  3. 官方快速入门示例命名：`BB_Enemy`、`BT_Enemy`、`EQS_FindPlayer`、`AIC_Enemy`、`EQC_PlayerContext`。

`EQC_PlayerContext`：给测试一个玩家参考点

创建 `EnvQueryContext_BlueprintBase` 蓝图，覆盖 `Provide Single Actor`，返回 `Get Player Character`。这样 `Trace` 和 `Distance` 能相对玩家进行评估。

EQC_PlayerContext
  Override Provide Single Actor
  Resulting Actor = Get Player Character

官方快速入门也提示：教程使用蓝图方便理解，更优化的方式可以用 C++ 创建 Context。

`Points: Grid`：围绕 AI 生成候选位置

在 `EQS_FindPlayer` 中从 `Root` 接出 `Points: Grid`。示例设置：

属性值含义
`GridHalfSize``800`控制网格范围的一半。
`Space Between``150.0`控制点之间间隔，也影响候选点数量。
`Generate Around``Querier`围绕 AI 生成，而不是围绕玩家生成。

两个测试：先能看见，再按距离选

测试关键设置作用
`Trace``Test Purpose = Filter Only`
`Context = EQC_PlayerContext`
`Bool Match = Disabled`		过滤掉无法看到玩家的点。
`Distance``Test Purpose = Score Only`
`Scoring Factor = -1.0`		对剩余点按距离计分；示例中 `-1.0` 倾向更近的有效点。

`Run EQS Query` 写黑板

在行为树战斗分支中添加 `Run EQS Query`，设置 `Query Template = EQS_FindPlayer`，把结果写入 `MoveToLocation`，再执行 `Move To`。

Run EQS Query
  Query Template = EQS_FindPlayer
  Blackboard Key = MoveToLocation
Move To
  Destination = MoveToLocation

看见系统在想什么

  • 运行时按撇号键 `'` 打开 AI Debug；`1` 通用 AI、`2` 行为树、`3` EQS、`4` AI 感知。
  • `EQS Testing Pawn` 可在编辑器中指定 `Query Template` 并查看调试球体。
  • 调试信息会显示网格点、分数和值，以及被选中的 `Winner`。

6. 互动调参：过滤、计分、Run Mode 会怎样改变结果？

这个小实验模拟 `Trace` 先过滤、`Distance` 再计分的效果。数据是教学用的简化候选点，不代表引擎内部完整算法。

性能直觉：`GridHalfSize` 越大、`Space Between` 越小，候选点通常越多，测试成本越高。官方快速入门在设置网格时明确提醒：过大的网格可能影响游戏性能。

7. 排错决策树：新人最常卡在哪里？

创建不了 Environment Query

内容浏览器里找不到 EQS 资源。

查询没有结果

Run EQS 后黑板没写入目标。

AI 跑向奇怪位置

点太远、跑过玩家或一直绕路。

看不到调试球体

不知道哪些点被过滤或选中。

创建不了 Environment Query

先检查：`Project Settings > Plugins` 是否启用了 `Environment Query Editor`。

下一步：启用后再从 `Artificial Intelligence` 创建 `Environment Query` 资源。

编辑器调试注意：官方相关文档提到 `EQSTestingPawn` 可能给处理器造成较大负担，对活跃查询模板进行修改可能导致长时间挂起；编辑时尽量明确查询模板并控制候选规模。
